# 项目亮点

- 极简高颜值（颜值即正义）
- 支持设置多个备选志愿（多设置几个备选方案，总是有备无患）
- 内置一个可以用来自动识别选课验证码的 tensorflow 模型，准确率高达 90% 以上
- 强大到变态的志愿筛选系统（不仅可以指定课程，授课教师，还可以筛选上课时间，地点甚至可以精确到 xx 教学楼的 xx 层楼 xx 间教室）

# 幕后花絮

Killer 是我做过的第一个 APP，也是我正式开始自学前端开发的起点。

Killer 这个名称是指，它就像是一个选课杀手一样，能够帮你成功选到自己心仪的课程，所向无敌。

这个项目的前身是一个 Python 的选课脚本，目的是为了解决我们学校教务系统卡顿，很难通过常规手段选到心仪课程的难题。

最初这个项目，只是一个相对简陋的 python 脚本，只是拿来私用为自己一个人和关系比较好的舍友选课。

后来，我喜欢上一位法语系的小姑娘，像做一个能在她手机上安装和使用的 APP，帮她也选上想要的课程，然后便有了这个项目。

有趣的是，当时这个项目我在自己的公众号上，向同学宣传过，后来不知道是哪位无聊的小可爱，因为自己没选到课，向学校教务处举报有人通过计算机程序扰乱选课秩序。

然后当时，我还差点被学校教务处请去“喝茶”，所以从那以后，我将这个项目对外终止维护了。

比较讽刺的是，那次选课学校教务处破天荒的，在原来的选课流程上添加了验证码，

由于验证码过于变态（字符又是扭曲变形，又是加入各种干扰线和噪点），很多同学好不容易进到选课页面，却因为验证码填错，错过了选课的机会，这也为学校教务处招来了一片骂声。

这次变化对于 Killer 本身也是一个致命的打击，导致原来的选课脚本完全失效了，所以在后来的几个版本中，我也为 Killer 补齐了自动识别验证码的能力。

我先是根据教务系统的验证码特征，找到了生成对应验证码的 Java 库和近似参数，批量生成了 20000 张带标签的验证码，然后训练了一个 3 层卷积池化结构的简易 CNN 模型。

模型训练好之后，我把模型量化成适合在端侧部署的 TensorFlow Lite 格式，实现了在端侧离线本地识别验证码，最终识别率达到了 90% 以上，远高于人类的平均准确率。

后来这段验证码识别技术的研究过程，也被作为研究选题，写进了我的本科毕业论文里，算是非常有趣的一段经历。
